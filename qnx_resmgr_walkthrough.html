Building /dev/example:
from skeleton to a working device

A step-by-step walkthrough of how example_initialization.c, example_read.c, and example_write.c progressively build into the final example.c — plus an animated trace of what actually happens inside the resource manager when a client opens, reads, and writes the device.

1.

The four moving parts

Every QNX resource manager is built from the same framework pieces, regardless of what device it implements.

dispatch_t

Created by dispatch_create_channel(). Owns the channel that clients connect to and the internal registry of handlers.

connect_funcs / io_funcs

Function-pointer tables. iofunc_func_init() fills them with sane POSIX defaults; you override only the ones you care about — here, open, read, write.

iofunc_attr_t

Describes the device node itself: type (S_IFCHR), permissions (0666), owner. Used by the default handlers for permission checks.

dispatch_context_t

The per-loop receive buffer. dispatch_block() blocks here waiting for the next message or pulse — this is the MsgReceive() from the client/server diagram.

2.

Building it in three stages

The uploaded files aren't four unrelated programs — they're the same device built incrementally. Each stage adds one capability on top of the last.

example_initialization.c — registers /dev/example and wires up the framework, but the I/O handlers are stubs. io_read() always replies with 0 bytes (immediate EOF); io_write() accepts any number of bytes and claims success without storing anything. Nothing is broken — every syscall the client makes returns cleanly — there's just no real data path yet. 
int io_read (resmgr_context_t *ctp, io_read_t *msg, RESMGR_OCB_T *ocb)
{
    int status;
    if ((status = iofunc_read_verify(ctp, msg, ocb, NULL)) != EOK)
        return status;
    if ((msg->i.xtype & _IO_XTYPE_MASK) != _IO_XTYPE_NONE)
        return ENOSYS;

    // reply with 0 bytes -- that is EOF, no real data yet
    MsgReply(ctp->rcvid, 0, NULL, 0);

    if (msg->i.nbytes > 0)
        ocb->attr->flags |= IOFUNC_ATTR_ATIME;
    return _RESMGR_NOREPLY;
}

int io_write (resmgr_context_t *ctp, io_write_t *msg, RESMGR_OCB_T *ocb)
{
    int status;
    if ((status = iofunc_write_verify(ctp, msg, ocb, NULL)) != EOK)
        return status;
    if ((msg->i.xtype & _IO_XTYPE_MASK) != _IO_XTYPE_NONE)
        return ENOSYS;

    // pretend we wrote everything -- data actually goes nowhere
    MsgReply(ctp->rcvid, msg->i.nbytes, NULL, 0);

    if (msg->i.nbytes > 0)
        ocb->attr->flags |= IOFUNC_ATTR_MTIME | IOFUNC_ATTR_CTIME;
    return _RESMGR_NOREPLY;
}
example_read.c — io_read() now returns real data: a fixed string "hello ", truncated to whatever the client actually asked for via min(nb, msg->i.nbytes). io_write() is untouched — still the stub from stage 1. 
int io_read (resmgr_context_t *ctp, io_read_t *msg, RESMGR_OCB_T *ocb)
{
    int status;
    char data[] = "hello "; // made up data
    size_t nb;

    if ((status = iofunc_read_verify(ctp, msg, ocb, NULL)) != EOK)
        return status;
    if ((msg->i.xtype & _IO_XTYPE_MASK) != _IO_XTYPE_NONE)
        return ENOSYS;

    // how much data do we have, vs. what the client wants?
    nb = sizeof(data);
    nb = min( nb, msg->i.nbytes );

    // copy back to client, tell their read() to return nb
    MsgReply(ctp->rcvid, nb, data, nb);

    if (nb > 0)
        ocb->attr->flags |= IOFUNC_ATTR_ATIME;
    return _RESMGR_NOREPLY;
}

/* io_write() unchanged from stage 1 -- still a stub */
example_write.c (== final example.c) — io_write() now actually captures the client's bytes: allocate a buffer, pull the payload out with resmgr_msgget(), and print it to stdout. io_read() stays as it was in stage 2. 
int io_write (resmgr_context_t *ctp, io_write_t *msg, RESMGR_OCB_T *ocb)
{
    int status;
    ssize_t nb;
    char *buf;

    if ((status = iofunc_write_verify(ctp, msg, ocb, NULL)) != EOK)
        return status;
    if ((msg->i.xtype & _IO_XTYPE_MASK) != _IO_XTYPE_NONE)
        return ENOSYS;

    if (msg->i.nbytes == 0) {
        MsgReply(ctp->rcvid, 0, NULL, 0);
        return _RESMGR_NOREPLY;
    }

    // somewhere to put the data -- in a real driver, a hardware buffer
    buf = malloc(msg->i.nbytes);
    if (buf == NULL) return ENOMEM;

    // pull the payload out of the message, past our header
    nb = resmgr_msgget(ctp, buf, msg->i.nbytes, sizeof(*msg));
    if (nb == -1) { free(buf); return errno; }

    // dump it to stdout instead of into the void
    status = write( STDOUT_FILENO, buf, nb );
    free(buf);
    if (status == -1) return errno;

    MsgReply(ctp->rcvid, nb, NULL, 0);

    if (nb > 0)
        ocb->attr->flags |= IOFUNC_ATTR_MTIME | IOFUNC_ATTR_CTIME;
    return _RESMGR_NOREPLY;
}
